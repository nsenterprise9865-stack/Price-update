Project Proposal

Grocery Price Comparison System

Analysis of Price Synchronisation Methodologies & Strategic Roadmap

Executive Overview

This document details the architectural choices for maintaining real-time pricing data across various grocery networks (Shwapno, Agora, Meena Bazar). Our goal is to balance reliability, cost-efficiency, and speed to market.

Option 1: CSV Bulk Administration

A controlled update method where the system processes structured data files provided or prepared by the administrator. This is the industry-standard starting point for MVP launches.

graph LR data[Price Sources] --> admin[Weekly CSV Batch] admin --> upload[System Validator] upload --> db[(Production DB)] db --> live[User Comparison App] style admin fill:#eff6ff,stroke:#3b82f6
  • Absolute Accuracy: Multi-point verification ensures that the prices users see are 100% correct.
  • Infrastructure Stability: Eliminates dependency on third-party website layouts which change frequently.
  • Zero Downtime: Uploads are processed in milliseconds without hindering site performance.

Option 2: Automated Cloud Scraping

A sophisticated "Crawler" engine that autonomously monitors supermarket digital storefronts and pushes updates to your platform via background workers.

graph TD sites[Store Digital Fronts] --> engine[Crawler Engine] engine --> parser[AI Parser] parser --> api[Data API] api --> db[(Production DB)] style engine fill:#f0fdf4,stroke:#10b981
  • Scalable Automation: Capable of tracking 10,000+ SKU items without human intervention.
  • Real-time Sync: Can detect flash sales or hourly price drops automatically.
  • Advanced Engineering: Requires continuous monitoring and script adjustments when stores update their security layers.

Scalability & Security

Future-Proof Growth

The system is designed to scale from **3 stores to 30+ stores** without significant architectural changes. AI-Enhanced scraping allows us to add new stores in hours rather than days.

Security & Compliance

We use high-quality **Residential Proxies** to ensure our automated robots act like regular customers. This prevents IP blocking and ensures 99.9% uptime for your price data.

Project Implementation Roadmap

Phase 1: Foundation (Current)

Setup the comparisons platform and CSV bulk upload for immediate stability.

Phase 2: Automation Setup (Month 1-2)

Integration of AI Scraping APIs and Daily Background Sync for all major supermarkets.

Phase 3: Intelligent Analytics (Future)

AI-driven price prediction to alert users when a price is likely to drop.

Strategic Recommendation

For the initial Phase 1 (Go-to-Market), we strongly advise Option 1 (CSV Bulk Administration). It provides the highest level of stability and trust for new users. As the platform scales, Option 2 can be integrated as a backend performance upgrade without disrupting the existing user base.
